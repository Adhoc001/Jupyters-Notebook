Check that group list is not empty.

NOTE

An empty list means World access, which is probably not the intention for custom kernels. Only kernels provided by Notebooks Team should normally provide World access.

2

Check if the name field of an already registered kernel was not modified.

  1. The name field of an already registered kernel should not be modified as changing it makes the registration flow believe that you want to register a new kernel and creates a new entry for it. This removes the old entry and any kernel spec pointing to it.
  2. The displayName, can be changed at any time without any adverse impact.
NOTE

THIS IS FOR EXISTING KERNELS ONLY!
